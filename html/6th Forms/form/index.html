  • The term 'form' has referred to a printed document that contains spaces for you to fill in information.
  • HTML borrows the concept of a form to refer to different elements that allow you to collect information from visitors to your site.
  • The best known form on the web is probably the search box that sits right in the middle of Google's homepage.
  • Forms also allow users to perform other functions online.
  • HTML forms give you a set of elements to collect data from your users.

  • Text input (single-line) Used for a single line of text such as email addresses and names.
  • Password input Like a single line text box but it masks the characters entered.
  • Text area (multi-line) For longer areas of text, such as messages and comments.
  • Radio buttons For use when a user must select one of a number of options..
  • Checkboxes When a user can select and unselect one or more options
  • Drop-down boxes When a user must pick one of a number of options from a list.
  • Multiple Select Box When a user pick one and more number of options from a list.
  • Submit buttons To submit data from your form to another web page.
  • Image buttons Similar to submit buttons but they allow you to use an image..
  • File upload Allows users to upload files (e.g. images) to a website.
  • How Forms Work.
    1. A user fills in a form and then presses a button to submit the information to the server.
    2. The name of each form control is sent to the server along with the value the user enters or selects.
    3. The server processes the information using a programming language such as PHP, C#, VB.net, or Java. It may also store the information in a database.
    4. The server creates a new page to send back to the browser based on the information received.
  • The<form> element defines an HTML form:.
  • <form>This element should always carry the action attribute and will usually have a method and id attribute too.
  • action Specifies an address (url) where to submit the form (default: the submitting page).
  • method Specifies the HTTP method used when submitting the form (default: GET).
  • The get method the values from the form are added to the end of the URL specified in the action attribute.
  • The get method is ideal for:
    1. short forms (such as search boxes)
    2. when you are just retrieving data from the web server (not sending information that should be added to or deleted from a database).
  • The post methodvalues are sent in what are known as HTTP headers. As a rule of thumb you should use the post method if your form.
    1. allows users to upload a file.
    2. it is very long.
    3. contains sensitive data (e.g. passwords).
    4. adds information to, or deletes information from, a database.
  • If the method attribute is not used, the form data will be sent using the get method
  • id the value is used to identify the form distinctly from other elements on the page (and is often used by scripts â€” such as those that check you have entered information into fields that require values).
  • The<input>element is the most important form element.
  • The<input>element has many variations, depending on the type attribute.
  • <input type="text"> defines a one-line input field for text input.
  • each form control requires a name attribute. Specifies a name used to identify the form (for DOM usage: document.forms.name).
  • The size attribute should not be used on new forms. It was used in older forms to indicate the width of the text input (measured by the number of characters that would be seen).
  • The maxlength attribute to limit the number of characters a user may enter into the text field. Its value is the number of characters they may enter.
  • type="password" it creates a text box that acts just like a single-line text input, except the characters are blocked out.
  • The <textarea> element is used to create a mutli-line text input.
  • <textarea> this is not an empty element.
  • type="radio" Radio buttons allow users to pick just one of a number of options.type="radio" Radio buttons allow users to pick just one of a number of options.
  • value The value attribute indicates the value that is sent to the server for the selected option.
  • checked The checked attribute can be used to indicate which value (if any) should be selected when the page loads.
  • Once a radio button has been selected it cannot be deselected. The user can only select a different option. If you are only allowing the user one option and want them to be able to deselect it.
  • type="checkbox" Checkboxes allow users to select (and unselect) one or more options in answer to a question.
  • The<select> element is used to create a drop down list box
  • It contains two or more <option> elements.
  • The<option>element is used to specify the options that the user can select from.
  • selected The selected attribute can be used to indicate the option that should be selected when the page loads.
  • The function of the drop down
    1. If users need to see all options at a glance, radio buttons are better suited.
    2. If there is a very long list of options (such as a list of countries), drop down list boxes work better.
  • multiple You can allow users to select multiple options from this list by adding the multiple attribute with a value of multiple.
  • type="file" creates a box that looks like a text input followed by a browse button.
  • type="submit" The submit button is used to send a form to the server.
  • type="image"attribute use an image for the submit button.
